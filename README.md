# [JenkinsQA Automation Framework (Playwright + JS)](https://github.com/slavaroskoshnyy/JenkinsQA_JS_2025_fall.git)

**Учебный командный проект [RedRover.school](https://redrover.school)**.

Проект автоматизации тестирования UI для CI/CD инструмента **Jenkins**.
Реализован командой студентов на стеке **JavaScript + Playwright** с использованием паттерна Page Object Model (POM).

## О проекте

В рамках обучения мы командой разработали фреймворк для покрытия автотестами функциональности Jenkins.
**Ключевые цели:**

- Организация командной работы в Git (ветки, Pull Requests, Code Review).
- Построение масштабируемой архитектуры тестов.
- Реализация Page Object паттерна для сложного интерфейса.

## Особенности архитектуры

- **Page Object Model (POM):** Логика взаимодействия со страницами вынесена в отдельную директорию `POM/`, что делает тесты читаемыми и легкими в поддержке.
- **Хелперы:** В папке `helpers/` реализованы вспомогательные функции для генерации тестовых данных и общих действий.
- **Глобальный сетап:** Использование `global-setup.js` для предварительной настройки окружения.
- **Конфигурация:** Гибкая настройка через `playwright.config.js`.

## Стек технологий

- **Playwright** — основной инструмент автоматизации.
- **JavaScript (Node.js)** — язык написания тестов.
- **Prettier / ESLint** — линтинг и форматирование кода.
- **Dotenv** — управление переменными окружения.

## Структура проекта

├── POM/ # Классы Page Object (страницы Jenkins) </br>
├── helpers/ # Вспомогательные функции (генераторы данных, утилиты)</br>
├── tests/ # Сценарии тестирования (Specs)</br>
├── global-setup.js # Скрипт глобальной настройки</br>
├── playwright.config.js # Конфигурация Playwright</br>
└── package.json # Зависимости и скрипты запуска

## Как запустить локально

1. **Клонируйте репозиторий:**

`git clone https://github.com/slavaroskoshnyy/JenkinsQA_JS_2025_fall.git` </br>
`cd JenkinsQA_JS_2025_fall`

2. **Установите зависимости:**

`npm install` </br>
`npx playwright install`

3. **Настройте окружение:**

Создайте файл `.env` (по примеру `.env_example`) и укажите URL и учетные данные тестового инстанса Jenkins.

4. **Запустите тесты:**

`npx playwright test`
